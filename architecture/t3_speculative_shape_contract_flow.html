T3 Speculative Prototype Flow

This is the current in-progress speculative wrapper around scheduled multilingual T3: shared prefill, draft proposal, verifier block, accept-or-correct commit, and the unchanged S3 boundary.
The currently validated mode is self-draft + greedy + cfg_weight=0.5 + alignment guard off. Dimension labels beside each tuple are semantic mnemonics, not code variable names.

What This Proves
  • The speculative wrapper can sit around the current scheduled T3 decode loop.
  • Prefill, draft, verify, and commit boundaries are shape-correct.
  • Causal verifier alignment is correct for greedy self-draft mode.
  • Accepted block length advances the real verifier cache by exactly +K.
  • Committed speculative tokens matched baseline greedy tokens exactly.
The current strongest correctness signals are exact_token_match = True, correction_tokens_total = 0, and matching rendered waveform length.
What This Does Not Yet Prove
  • No real latency win yet.
  • No GPU-saturation win yet.
  • No concurrency win yet.
  • No evidence yet for an external smaller draft model.
  • Self-draft is still a correctness scaffold, not a deployable serving strategy.
Fair repeated runs currently show the opposite of a speedup: baseline about 1.68s vs speculative about 2.12s, plus higher peak allocation.
