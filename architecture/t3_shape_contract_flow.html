After prefill, every later decode round sends exactly one new speech-token embedding back through the transformer with KV cache. One sampled token is embedded as (1, 1, 1024) (logical_request_rows, decode_step_token_len, hidden_size) and then duplicated back to CFG width. The first traced cached step confirms the expected cache growth from sequence length 72 to 73.
Where Speculation Would Fit
Inside this one-token cached decode loop, not outside the request boundary. The outer contract should stay: same request in, same speech tokens out. The remaining risk is no longer tensor shape; it is verifier/logit alignment when validating a proposed token block against the causal target model.
