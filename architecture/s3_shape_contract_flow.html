Token-to-mel contract
(1, 85) -> (1, 80, 170)
Filtered speech tokens from `T3` map to 170 mel frames, which go into HiFT. Hydra runtime ensures tokens represent the current best planner output (k3).
Vocoder contract
(1, 80, 170) -> (1, 81600)
HiFT sees a zero-length cache and always returns a full waveform sample tensor. GPU-local cache stays empty in the synchronous scheduled path.
Strongest performance read
c2 is the `S3` sweet spot
Throughput peaks at c2; c4/c8 show the downstream scaling stress that Hydra+scheduled T3 still feeds into.

Unlike T3, S3 does not mutate reusable shared inference state per request. The risk is performance: `stage_s3_s` balloons past `c4` while inputs remain sequential. Hydra tokens arrive cleanly, so the next step is profiling the flow vs the HiFT path to decide if `S3` needs staging or batching similar to `T3`.
